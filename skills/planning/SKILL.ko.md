# 구현 계획 작성

<HARD-GATE>
plan.md를 작성하기 전에 설계가 승인되었는지 확인하십시오. 승인되지 않은 경우 먼저 brainstorming 스킬을 사용하십시오.
</HARD-GATE>

## 계획 작성 프로세스

### 1단계: plan.md 파일 생성
프로젝트 루트에 `plan.md` 파일을 생성합니다:

```markdown
# 구현 계획: [기능명]

## 목표
[달성하려는 것]

## 완료 기준
- [ ] [검증 가능한 기준 1]
- [ ] [검증 가능한 기준 2]

## 구현 단계
- [ ] 1. [구체적인 단계]
- [ ] 2. [구체적인 단계]
- [ ] 3. [구체적인 단계]

## 검증 계획
- [ ] 테스트 실행: `[테스트 명령어]`
- [ ] 린터 실행: `[린터 명령어]`
- [ ] 수동 확인: [확인 방법]

## 영향 받는 파일
- `[파일 경로]`: [변경 이유]
```

### 2단계: 계획 검토
- 각 단계가 독립적으로 검증 가능한지 확인합니다.
- 단계가 너무 크면 더 작게 나눕니다.
- 의존성이 올바른 순서로 되어 있는지 확인합니다.

### 3단계: 실행 시작
계획이 완성되면 단계별로 실행합니다. 각 단계 완료 후 체크박스를 표시합니다.

## 핵심 규칙
- **반드시**: 코드 작성 전 plan.md 작성
- **반드시**: 각 단계는 검증 가능해야 함
- **절대 금지**: 여러 단계를 동시에 진행
