# 코드 검토 및 검증

<HARD-GATE>
테스트가 통과하고 린터가 오류 없이 실행되기 전까지 작업을 "완료"로 선언하지 마십시오.
</HARD-GATE>

## 검토 체크리스트

### 기능 검증
- [ ] 모든 완료 기준이 충족되었습니까?
- [ ] 엣지 케이스가 처리되었습니까?
- [ ] 에러 처리가 적절합니까?

### 코드 품질
- [ ] 코드가 DRY 원칙을 따릅니까? (반복 없음)
- [ ] 함수/메서드가 단일 책임을 가집니까?
- [ ] 변수/함수 이름이 명확하고 설명적입니까?
- [ ] 복잡한 로직에 주석이 있습니까?

### 테스트
- [ ] 단위 테스트가 작성되었습니까?
- [ ] 모든 테스트가 통과합니까? (`npm test` / `pytest` 등)
- [ ] 테스트 커버리지가 충분합니까?

### 보안 및 성능
- [ ] 보안 스킬의 체크리스트를 통과했습니까?
- [ ] 명백한 성능 문제가 없습니까?
- [ ] 불필요한 API 호출이나 데이터베이스 쿼리가 없습니까?

### 문서화
- [ ] `handoff.md`가 업데이트되었습니까?
- [ ] 새로운 API나 함수에 문서가 있습니까?
- [ ] 까다로운 버그가 `docs/wiki/gotchas.md`에 기록되었습니까?

## 핵심 규칙
- **절대 금지**: 테스트 실패 상태로 작업 완료 선언
- **반드시**: handoff.md 업데이트
- **반드시**: 린터 오류 없이 통과
