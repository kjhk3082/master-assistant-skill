# 아이디어를 설계로 발전시키기

<HARD-GATE>
코드를 작성하기 전에 반드시 이 스킬을 사용해야 합니다. 추측하거나 즉시 구현하지 마십시오.
</HARD-GATE>

## 브레인스토밍 프로세스

### 1단계: 의도 파악
다음 사항이 명확하지 않으면 사용자에게 질문하십시오:
- **목표**: 이 기능/변경이 달성하려는 것은 무엇입니까?
- **사용자**: 누가 이것을 사용합니까? 어떤 맥락에서 사용합니까?
- **제약 조건**: 기술적 제약, 디자인 제약 또는 비즈니스 규칙이 있습니까?
- **완료 기준**: "완료"는 어떤 모습입니까?

### 2단계: 옵션 탐색
최소 2-3가지 접근 방식을 제안합니다:
- 각 옵션의 장단점을 간략히 설명합니다.
- 권장 접근 방식과 그 이유를 명시합니다.
- 명백히 잘못된 방향은 제외합니다.

### 3단계: 설계 승인 받기
진행하기 전에 반드시 사용자의 명시적 승인을 받아야 합니다:
```
제안된 접근 방식: [접근 방식 요약]
진행해도 될까요?
```

### 4단계: planning 스킬로 전환
승인을 받으면 `planning` 스킬을 호출하여 단계별 구현 계획을 작성합니다.

## 핵심 규칙
- **절대 금지**: 승인 없이 코드 작성 시작
- **절대 금지**: 요구사항 추측
- **반드시**: 불명확한 사항은 질문으로 해결
