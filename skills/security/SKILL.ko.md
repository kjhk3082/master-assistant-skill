# 보안 코딩 관행

<HARD-GATE>
사용자 입력을 직접 SQL 쿼리, 셸 명령어 또는 HTML에 삽입하지 마십시오.
</HARD-GATE>

## 보안 체크리스트

### 입력 유효성 검사
- [ ] 모든 사용자 입력을 검증하고 정제합니다.
- [ ] 매개변수화된 쿼리 또는 ORM을 사용합니다 (SQL 인젝션 방지).
- [ ] 출력을 HTML 인코딩합니다 (XSS 방지).
- [ ] 파일 업로드를 검증합니다 (유형, 크기, 내용).

### 인증 및 권한 부여
- [ ] 비밀번호를 bcrypt/argon2로 해시합니다 (MD5/SHA1 금지).
- [ ] JWT 토큰을 올바르게 검증합니다.
- [ ] 모든 엔드포인트에 권한 부여 검사를 구현합니다.
- [ ] 속도 제한을 구현합니다.

### 민감한 데이터
- [ ] 비밀번호, API 키를 로그에 기록하지 않습니다.
- [ ] 환경 변수를 사용합니다 (코드에 하드코딩 금지).
- [ ] 전송 중 데이터를 암호화합니다 (HTTPS).
- [ ] 저장 시 민감한 데이터를 암호화합니다.

### 의존성
- [ ] 알려진 취약점이 있는 패키지를 사용하지 않습니다.
- [ ] 의존성을 최신 상태로 유지합니다.

## 핵심 규칙
- **절대 금지**: 사용자 입력을 직접 쿼리/명령어에 삽입
- **절대 금지**: 민감한 데이터를 코드에 하드코딩
- **반드시**: 모든 외부 입력을 신뢰하지 않는 것으로 처리
