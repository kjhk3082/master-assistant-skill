# UI/UX 설계 및 구현

<HARD-GATE>
임의의 색상, 간격 또는 폰트를 사용하지 마십시오. 항상 프로젝트의 디자인 시스템을 따르십시오.
</HARD-GATE>

## UI/UX 체크리스트

### 디자인 시스템 준수
- [ ] 프로젝트의 색상 팔레트를 사용합니다 (임의 색상 금지).
- [ ] 4px 그리드 시스템을 따릅니다 (간격: 4, 8, 12, 16, 24, 32px).
- [ ] 프로젝트의 타이포그래피 스케일을 사용합니다.
- [ ] 기존 컴포넌트를 재사용합니다 (새로 만들기 전에 확인).

### 접근성 (WCAG 2.1 AA)
- [ ] 색상 대비 비율: 텍스트 4.5:1 이상, 대형 텍스트 3:1 이상.
- [ ] 모든 인터랙티브 요소에 키보드 포커스 표시기.
- [ ] 이미지에 의미 있는 alt 텍스트.
- [ ] 폼 필드에 연결된 레이블.

### 반응형 디자인
- [ ] 모바일 우선 접근 방식.
- [ ] 브레이크포인트: 모바일(320px), 태블릿(768px), 데스크톱(1024px).
- [ ] 터치 타겟 최소 44x44px.

### AI 냄새 제거
- [ ] 지나치게 완벽한 그라디언트 없음.
- [ ] 과도한 애니메이션 없음.
- [ ] 인간적인 불규칙성과 개성 추가.

## 핵심 규칙
- **절대 금지**: 디자인 시스템 없이 인라인 스타일 사용
- **반드시**: 모바일에서 먼저 테스트
- **반드시**: 접근성 검사 통과
