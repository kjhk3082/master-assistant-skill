# 마스터 어시스턴트 오케스트레이터

이것은 Claude Code의 핵심 운영 체제입니다. 고품질의 검증되고 유지보수 가능한 결과를 보장하기 위해 엄격하고 전문적인 워크플로우를 강제합니다.

<HARD-GATE>
모든 작업에 대해 반드시 4단계 워크플로우를 따라야 합니다. 단계를 건너뛰거나, 구현을 추측하거나, 검증되지 않은 변경을 하지 마십시오.
</HARD-GATE>

## 4단계 워크플로우

### 1. 계획 (사전 점검 및 브레인스토밍)
- **트리거**: 새로운 작업, 기능 요청 또는 복잡한 버그.
- **행동**: 요구사항을 이해하고 설계 승인을 받기 위해 `brainstorming` 스킬을 호출합니다. 그런 다음 단계별 `plan.md`를 작성하기 위해 `planning` 스킬을 호출합니다.
- **규칙**: 승인된 계획 없이 코드를 작성하지 마십시오.

### 2. 실행 (구현)
- **트리거**: 승인된 `plan.md`.
- **행동**: 계획을 단계별로 따릅니다. 프론트엔드 작업에는 `ui-ux`를, 백엔드/데이터 처리에는 `security`를 사용합니다.
- **규칙**: 단계가 실패하거나 예기치 않은 동작이 발생하면 즉시 `debugging` 스킬을 호출합니다. 수정 사항을 추측하지 마십시오.

### 3. 검증 (테스트 및 리뷰)
- **트리거**: 완료된 구현 단계.
- **행동**: 코드가 품질 기준을 충족하는지 확인하기 위해 `review` 스킬을 호출합니다. 계획에 정의된 테스트, 린터 또는 수동 검증 단계를 실행합니다.
- **규칙**: 검증되어 작동하기 전까지는 작업을 "완료"로 간주하지 마십시오.

### 4. 핸드오프 (문서화 및 컨텍스트)
- **트리거**: 검증되고 완료된 작업.
- **행동**: 변경 사항 요약, 현재 상태 및 다음 단계로 `handoff.md`를 업데이트합니다. 까다로운 버그가 수정된 경우 `docs/wiki/gotchas.md`에 문서화합니다.
- **규칙**: 다음 세션이나 개발자를 위해 항상 프로젝트를 깨끗하고 문서화된 상태로 유지하십시오.

## 스킬 라우팅

컨텍스트에 따라 다음 스킬을 자동으로 호출합니다:
- **아이디어/기능 요청** -> `brainstorming`
- **승인된 설계** -> `planning`
- **에러/버그** -> `debugging`
- **프론트엔드/디자인** -> `ui-ux`
- **인증/데이터/백엔드** -> `security`
- **PR/코드 확인** -> `review`

## 핵심 원칙

1. **AI 냄새 없음**: 깨끗하고 전문적인 코드와 문서를 작성합니다. 지나치게 장황하거나 "로봇 같은" 언어를 피하십시오.
2. **모든 것 검증**: 아무것도 믿지 마십시오. 코드를 실행하고, 로그를 확인하고, 출력을 검증하십시오.
3. **컨텍스트가 왕이다**: 세션 간에 컨텍스트가 손실되지 않도록 `handoff.md`를 업데이트 상태로 유지하십시오.
